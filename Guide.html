How the System Works

V Haus Living Delivery Management System — a complete guide for the whole team

📱

Telegram Bot

Salesmen submit orders, reschedule deliveries, and flag mistakes. Drivers report delivery results. Everything flows into the web app automatically.

🌐

Web App

Admin views all orders, plans daily delivery routes, assigns lorries, confirms schedules, and manages service orders — all in one place.

🗄️

Live Database

Everything is stored in real-time. Any update from the bot instantly appears in the web app and vice versa.

🔄 Full System Flow

📷 Salesman snaps sales order photo → sends to Telegram bot
↓
🤖 AI reads the photo and extracts all order details automatically
↓
✅ Salesman confirms or corrects the details → order saved
↓
🗓️ Admin opens web app → assigns order to lorry route → confirms schedule
↓
🚛 Driver sends delivery report via Telegram after each delivery
↓
✅ Order marked Delivered — or service pending created if not settled

👥 Who Uses What

Salesman — Telegram Bot (Group A)

Everything salesmen need to know about using the bot

💡
Send any message to the bot to see the main menu. Type cancel at any step to go back.

📋 Main Menu

Bot: 👋 Hello! Welcome to V Haus Living (PG) Bot

Reply with a number to get started:

1️⃣ New Order — Send a sales order photo
2️⃣ Reschedule — Change a delivery or service date
3️⃣ Flag Wrong Order — Report incorrect order info
4️⃣ Help — Show this menu

1️⃣ Submitting a New Sales Order

1
Type "1" or "new order"
Bot enters New Order mode and asks you to send a photo.
2
Take a clear photo of the sales order
Make sure the SO number, customer name, delivery date, and items are all visible. Good lighting, no shadows.
3
AI extracts all details
Bot reads the photo and shows you a preview of what it found. Check all details carefully.
4
If wardrobe or fitting detected — bot asks for trip count
Bot will ask: "How many trips are needed?" Reply with a number (e.g. 3). This sets up multiple delivery trips automatically.
5
Confirm or correct
Reply YES to save, CANCEL to discard, or describe what needs changing.

How to correct details naturally

// Just type what needs changing — no special format needed
You: delivery date is 15/7
You: balance is 3840
You: item 2 is queen size
You: remove item 3
You: add dining table 6 seater

Delivery date formats the bot understands

You typeBot reads as
15/7 or 15/7/202615 July 2026
TMR or TomorrowNext day
ASAP21 days from order date
August or Aug1 August (guideline date)
Aug - Sept1 August (earliest in range)
TBC or TBDNo date — appears in TBC group

2️⃣ Rescheduling a Delivery

1
Type "2" or "reschedule"
Bot asks which SO you want to reschedule.
2
Enter the SO number
For a single delivery: just type the SO number, e.g. 11576
For a multi-trip order: include the trip number, e.g. 11576 Trip 2
3
Bot shows current scheduled date
Confirm you have the right order before entering the new date.
4
Enter the new date
Formats accepted: 15/7, 15/7/2026, tmr, TBC
⚠️
2-day rule: If you reschedule to a date that is within the next 2 working days AND the route is already confirmed, the system will send an approval request to the Operation Manager. You will be notified once approved or rejected.

3️⃣ Flagging a Wrong Order

Use this when you realise an order was saved with incorrect information (wrong balance, wrong customer name, wrong items, etc.)

1
Type "3" or "flag"
Bot asks which SO has wrong information.
2
Enter the SO number
e.g. 11576
3
Describe what is wrong
e.g. "balance should be 3840 not 1650" or "wrong customer name, should be Rebecca Tan"

Admin will be notified immediately and will fix the order in the web app.

StatusWhat it meansWhat happens

no settle Delivery not completed (e.g. item issue, customer not home) Service Pending created. Admin notified. For multi-trip: next trip stays scheduled.

Settled delivery

⚠️
Common mistakes:
• Status must be exactly settle or no settle
• SO number must match what is in the system
• Date format: 9/6 or 9/6/2026
• First line must be DELIVERY

🔄 Multi-Trip Orders

For wardrobe and large fitting orders that need multiple visits, the bot tracks each trip automatically.

SituationWhat bot does
Trip 1 — no settle, more trips remainingTrip 1 marked Completed. Next trip stays Scheduled.
Last trip — no settleAuto-creates one more trip (e.g. Trip 4). Admin notified to schedule it.
Any trip — settleOrder marked Delivered. All remaining trips cancelled.
💡
Commission note: Only Trip 1 counts for delivery commission. Trips 2, 3, etc. are follow-up visits and do not earn additional commission.

Web App — Service Orders

Manage all service and follow-up visits

Service orders are created when a delivery is not settled and converted from Service Pending. Each card shows:

  • SV number (e.g. SV-001) — the service order reference
  • ↩ Original SO — link back to the original delivery order
  • Service Date — when the service visit is scheduled (TBC if not yet set)
  • Service Note — what needs to be done
  • Service Trips — for multi-visit jobs, shows each trip with status and date

The service order is now ready to assign to a route.

Web App — Delivery Schedule

Plan daily lorry routes and assign orders to vehicles

🗓️ Daily Workflow

1
Select the date
Use the date picker at the top to choose the delivery date you want to plan.
2
Add a route
Click "+ Add Route". Select a vehicle from the list or enter lorry plate and driver name manually.
3
Assign orders to the route
Drag orders from the unassigned panel (left side) onto a route card. Or use the "Assign to route" dropdown on each order card.
4
Set scheduled times
For each order in the route, set the scheduled time (e.g. 10:00 - 12:00). Click "Use" to copy from the customer's preferred time.
5
Confirm the route
Change route status from Pending → Confirmed. This locks the route. Contact customers with their delivery times.
6
On delivery day — mark Out for Delivery
This can only be done on the actual delivery date. Updates all orders in the route to "Out for Delivery".

🔒 Route Status & Locking

StatusCan edit?When to use
Pending✅ Full accessWhile still planning and adjusting
Confirmed❌ Locked (revert to Pending to edit)After contacting customers with delivery time
Out for Delivery❌ Fully lockedOn the day when lorry departs
Delivered❌ Fully lockedAfter all deliveries completed
⚠️
To unlock a Confirmed route, change the status back to Pending. Remember to re-confirm and re-contact customers if you make changes.

Web App — Service Pending

Deliveries that were not settled — awaiting follow-up action

When a driver reports no settle for a single-trip order, it creates a Service Pending record here. Admin reviews each one and decides what to do.

🔄 Options

✅

Convert to Service Order

Creates a new Service Order with an SV number (SV-001, SV-002…). The order moves to the Service tab where admin can schedule the follow-up visit.

🗑️

Remove

Dismiss the record if the issue was resolved directly (e.g. customer came to collect the item) or is not applicable.

Converting to Service Order

1
Click "→ Convert to Service"
A confirmation dialog appears.
2
Add an admin remark (optional)
e.g. "Missing 1 pillow — need to replace" or "Wardrobe door hinge broken"
3
Click ✅ Confirm Convert
Service order created with SV number. Go to the Service tab to set the visit date and schedule it.
💡
The Convert button is disabled while processing to prevent creating duplicate service orders.

Tips & Important Notes

Things to know for smooth day-to-day use

📷 Photo Tips (Salesmen)

  • Place the order flat on a surface — avoid holding it in the air
  • Good lighting — avoid shadows covering the SO number or delivery date
  • Make sure the whole order is in frame including REMARKS section
  • One order per photo — do not combine multiple orders in one image
  • If AI extracts wrong info, just correct it by replying naturally — no need to resend the photo

🔁 One Draft at a Time

⚠️
After sending an SO photo, you must confirm YES or CANCEL before sending another photo. If you send a second photo while a draft is active, your text replies will go to the active draft instead.

🔒 Duplicate SO Numbers

⚠️
The system will not allow saving an order if the SO number already exists. If the bot says "SO already exists", do not resend — instead check with admin whether the order was already saved.

📅 Working Days Rule

Saturday counts as a working day. Sunday is the only rest day. The 2-day approval rule applies to reschedules within the next 2 working days (Mon–Sat) when a route is already confirmed.

💰 Commission Tracking

For multi-trip orders, only Trip 1 earns delivery commission. This is recorded as the first delivery date in the system. Trips 2, 3, etc. are follow-up visits with no additional commission.

🔄 Refreshing Data

The web app does not auto-refresh. Click the 🔄 Refresh button in the header or on individual tabs to load the latest data after any changes.
